# Dify Agent 配置指南

## 完整配置流程

### 第一步：创建应用

1. 登录 Dify 控制台
2. 点击"工作室" → "创建应用"
3. 选择"Agent" 类型
4. 应用名称：金蝶 ERP 智能助手
5. 图标和描述：自定义

---

### 第二步：配置系统提示词

在"提示词"区域输入：

```
你是金蝶ERP智能助手。

【可用工具】
- 获取金蝶技能文档: 获取指定技能的完整接口文档和调用示例
- 调用金蝶API: 调用金蝶WebAPI接口

【工作流程】
1. 收到用户问题后，判断需要使用哪个技能（kingdee_query 查询 或 kingdee_save 保存）
2. 若本次对话中已获取过该技能文档，直接使用已知格式，无需重复获取
3. 若未获取过该技能文档，先调用"获取金蝶技能文档"工具获取接口说明
4. 根据文档构造正确的请求体，调用"调用金蝶API"工具
5. 获得结果后用中文回答用户
6. 若查询结果为空，告知用户未找到相关数据
7. 用户输入均为中文，请按字面意思理解

【回答原则】
- 严格按用户问题作答，用户问什么就回答什么，不要延伸、不要补充额外信息
- 如果用户只问库存最多的物料，只回答那一个物料，不要列出前三或前五
- 如果用户只问某个仓库是否有某物料，只回答有或没有及数量，不要罗列其他仓库
- 回答尽量简洁，数据直接给出，不加冗余解释和总结段落
- 对于金蝶 API 返回的错误，用通俗语言解释给用户

【常用技能】
- kingdee_query: 查询金蝶数据（库存、订单、物料、客户、供应商等）
- kingdee_save: 保存金蝶单据（采购订单、销售订单等）
```

---

### 第三步：添加工具

#### 工具 1：获取金蝶技能文档

点击"添加工具" → "自定义工具"：

**基本信息：**
- 工具名称：`get_kingdee_skill_doc`
- 工具描述：`获取指定技能的完整接口文档，包含请求体格式、字段说明和调用示例。首次使用某技能前必须调用此工具。`

**API 配置：**
- 请求方法：`POST`
- URL：`http://localhost:8080/api/dify/skill/doc`
  （如果 Dify 和 API 服务不在同一台机器，改为实际地址）
- 请求头：
  ```
  Content-Type: application/json
  ```
- 请求体：
  ```json
  {
    "skillName": "{{skillName}}"
  }
  ```

**参数定义：**
- 参数名：`skillName`
- 类型：`string`
- 必填：是
- 描述：`技能名称，可选值: kingdee_query（查询）、kingdee_save（保存）`

**响应示例：**
```json
{
  "success": true,
  "doc": "# 金蝶单据查询接口\n\n## 请求体格式\n..."
}
```

---

#### 工具 2：调用金蝶 API

点击"添加工具" → "自定义工具"：

**基本信息：**
- 工具名称：`call_kingdee_api`
- 工具描述：`调用金蝶WebAPI接口。skillName 指定技能，requestBody 为接口请求体 JSON。调用前须先通过 get_kingdee_skill_doc 获取该技能的文档。`

**API 配置：**
- 请求方法：`POST`
- URL：`http://localhost:8080/api/dify/kingdee/call`
- 请求头：
  ```
  Content-Type: application/json
  ```
- 请求体：
  ```json
  {
    "skillName": "{{skillName}}",
    "requestBody": "{{requestBody}}"
  }
  ```

**参数定义：**

参数 1：
- 参数名：`skillName`
- 类型：`string`
- 必填：是
- 描述：`技能名称，例如: kingdee_query、kingdee_save`

参数 2：
- 参数名：`requestBody`
- 类型：`string`
- 必填：是
- 描述：`发送给金蝶WebAPI的请求体JSON字符串，格式参照 get_kingdee_skill_doc 返回的文档示例`

**响应示例：**
```json
{
  "success": true,
  "result": "{\"success\":true,\"total\":3,\"records\":[...]}"
}
```

### 第四步：配置知识库（可选）

如果想让 Agent 自动参考 SKILL 文档：

1. 点击"知识库" → "创建知识库"
2. 名称：金蝶 API 文档
3. 上传文件：
   - `kingdee_query/SKILL.md`
   - `kingdee_save/SKILL.md`
4. 分段设置：自动分段
5. 保存并处理

回到 Agent 配置：
- 点击"上下文" → "添加知识库"
- 选择刚创建的知识库
- 检索设置：
  - 召回模式：语义检索
  - Top K：3
  - 相似度阈值：0.5

### 第五步：测试 Agent

在右侧预览窗口测试：

**测试 1：简单查询**
```
用户：查询所有客户
```

预期流程：
1. Agent 调用 `get_kingdee_skill_doc(skillName="kingdee_query")`
2. Agent 根据文档构造请求体
3. Agent 调用 `call_kingdee_api(skillName="kingdee_query", requestBody="{...}")`
4. Agent 返回查询结果

**测试 2：复杂查询**
```
用户：查询腰托的库存，只看成品仓
```

**测试 3：保存单据**
```
用户：创建一个采购订单，供应商编码 SUP001，采购物料 MAT001，数量 100
```

---

### 第六步：发布应用

1. 点击右上角"发布"
2. 选择发布方式：
   - WebApp：生成独立的 Web 应用
   - API：通过 API 调用
   - 嵌入网站：生成嵌入代码

3. 如果选择 WebApp：
   - 复制访问链接
   - 可自定义界面主题色、Logo
   - 支持分享给团队成员

### 2. 添加下一步建议

在"功能"区域 → "下一步建议"：

```
- 查询即时库存
- 查看本月销售订单
- 创建采购订单
- 查询物料资料
```

### 3. 配置变量

如果需要动态配置金蝶服务器地址：

1. 点击"变量" → "添加变量"
2. 变量名：`kingdee_base_url`
3. 类型：文本
4. 默认值：`http://localhost:8080`
5. 在工具 URL 中使用：`{{kingdee_base_url}}/api/dify/skill/doc`

### 4. 添加审计日志

在"功能"区域 → "内容审查"：
- 开启输入审查：过滤敏感词
- 开启输出审查：确保回复合规

## 对比原系统

| 功能 | 原 Java 系统 | Dify 系统 |
|------|-------------|-----------|
| Agent 编排 | LangChain4j 代码 | Dify 可视化配置 |
| 工具调用 | Java 方法 | HTTP API |
| 对话记忆 | MessageWindowChatMemory | Dify 自动管理 |
| Web 界面 | 自己开发 | Dify 自带 |
| 部署 | 编译 JAR | Docker 一键部署 |
| 维护成本 | 需要 Java 开发 | 低代码配置 |

## 常见问题

### Q1: 工具调用失败
A: 检查金蝶 API 服务是否启动，URL 是否正确

### Q2: Agent 不调用工具
A: 检查系统提示词是否清晰，工具描述是否准确

### Q3: 返回结果格式混乱
A: 在系统提示词中明确输出格式要求

### Q4: 如何查看调用日志
A: Dify 控制台 → 日志 → 查看每次对话的详细调用链路
